Component Tree Architecture - Why Is It Better

  • A component has a well-defined public API
  • Component isolation
  • Limits the number of ways a component can be modified
  • Makes modifications predictable
  • Component's state is well known
  • Immutable Data - better performance and predictability

  • Component isolation
  • Limits the number of ways a component can be modified, makes this modification predictable
